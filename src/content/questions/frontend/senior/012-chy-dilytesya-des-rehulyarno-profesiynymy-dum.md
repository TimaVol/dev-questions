## Відповідь

Регулярність важливіша за платформу. Можливі канали: internal Slack/Confluence (tech notes після релізу), GitHub discussions у open source, dev.to/Medium (якщо пишете), Twitter/X threads про web platform, українські спільноти (DOU, Telegram-чати, LocalSoft). На співбесіді назвіть 1–2, де реально активні, і що конкретно дали: «в чаті X обговорили View Transitions — наступного тижня спробували в нашому SPA». Не перелічуйте 10 спільнот для галочки — краще показати impact: «написав post про наш підхід до error boundaries — 3 команди в компанії адаптували».

## Приклад

```markdown
**Що роблю:**
- Щотижневий #frontend-notes у Slack — 1 інсайт (bug, RFC, release note)
- DOU — коментую питання про кар'єру і стек (без токсичності)
- Contributing: 2 merged PR у docs Vite

**Що з цього вийшло:** колега з іншого відділу запитав про наш ADR template — уніфікували підхід.
```

## Юз кейси

- Knowledge sharing після інциденту: короткий write-up у wiki
- Менторство в спільноті: відповіді на питання junior у чаті
- Cross-pollination: ідеї з конференції → proposal на team retro
