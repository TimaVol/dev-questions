## Відповідь

Рекомендації мають бути особистими — «звідси я реально щось переніс у роботу». База для будь-якого фронтендера: MDN, web.dev, release notes браузерів і фреймворків. Подкасти: Syntax, JS Party, ShopTalk Show (CSS), React Podcast. Автори: Jake Archibald (service workers, perf), Lea Verou (CSS), Dan Abramov (React mental model — архів). Українське: DOU, IT Talks, канали про web perf. Не називайте 20 джерел — 3–4 з прикладом: «після епізоду Syntax про hydration mismatches — додали перевірку в CI».

## Приклад

```markdown
| Джерело          | Чому саме він                          |
|------------------|----------------------------------------|
| web.dev/vitals   | офіційні пороги CWV, без маркетингу    |
| Fireship (YT)    | швидкий огляд новинки перед spike      |
| Josh Comeau blog | глибокі статті про CSS/React           |
| Chrome DevTools docs | коли дебажу layout thrashing      |
```

## Юз кейси

- Онбординг junior: curated list «прочитай за перший місяць»
- Підготовка до співбесіди: свіжі release notes React/TS замість застарілих туторіалів
- Team learning: раз на 2 тижні — 30 хв «що нового в Baseline»
