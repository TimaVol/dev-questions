## Відповідь

**Переваги:** один мовний стек (JS/TS) на фронті й беку; неблокуючий I/O добре масштабується для багатьох одночасних з'єднань; велика екосистема npm; швидкий старт прототипів.

**Недоліки:** CPU-інтенсивні задачі блокують event loop; callback hell без дисципліни (хоча async/await це вирішує); слабша типізація без TypeScript; один поток за замовчуванням — потрібні worker threads для важких обчислень.

## Приклад

```js
// Добре: багато паралельних I/O-запитів
const results = await Promise.all(
  urls.map((url) => fetch(url).then((r) => r.json()))
);

// Погано: синхронне обчислення блокує сервер
function fib(n) {
  return n <= 1 ? n : fib(n - 1) + fib(n - 2);
}
```

## Юз кейси

- API-шар з високим навантаженням I/O (читання БД, зовнішні API)
- Мікросервіси з легкою логікою та частими мережевими викликами
- Уникати Node для відеокодування, ML-інференсу або важкої криптографії в головному потоці
